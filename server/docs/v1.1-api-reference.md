# LeanMate V1.1 后端接口文档

本文给开发、联调和人工验收使用，按接口逐个说明请求方式、URL、参数和返回结果。

机器可读契约仍以 [`../../docs/api/openapi.yaml`](../../docs/api/openapi.yaml) 为准。

## 基础信息

本地服务地址：

```text
http://localhost:8080
```

除登录和刷新 token 外，接口都需要在 Header 中携带访问令牌：

```json
{
  "Authorization": "Bearer <accessToken>"
}
```

统一成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

统一错误响应：

```json
{
  "code": 40101,
  "message": "未登录或登录已过期",
  "data": null
}
```

常见错误码：

| code | 含义 |
| --- | --- |
| 40001 | 参数错误 |
| 40101 | 未登录或 token 无效 |
| 40301 | 无权限访问资源 |
| 40401 | 资源不存在 |
| 40901 | 状态冲突 |
| 50001 | 服务端错误 |
| 50010 | AI 服务暂时不可用 |

## 1. 第三方登录

### 请求方式

```text
POST
```

### URL

```text
/v1/auth/oauth-login
```

### 是否需要登录

```text
否
```

### 参数

local/test 环境支持 mock Apple 登录。`identityToken` 格式为 `mock:<providerUserId>:<email>`。

```json
{
  "body": {
    "provider": "apple",
    "identityToken": "mock:apple-user-1:user@example.com",
    "authorizationCode": null,
    "deviceId": "local-test"
  }
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| provider | string | 是 | 当前支持 `apple`，`google` 保留但 V1.1 不支持 |
| identityToken | string | 是 | Apple identity token；本地可使用 mock token |
| authorizationCode | string | 否 | Apple 授权码，当前可为空 |
| deviceId | string | 否 | 设备标识，最长 128 |

### 返回结果

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "accessToken": "eyJhbGciOi...",
    "refreshToken": "refresh-token-value",
    "tokenType": "Bearer",
    "expiresIn": 3600,
    "user": {
      "id": "11111111-1111-1111-1111-111111111111",
      "nickname": null,
      "avatarUrl": null,
      "status": "active",
      "profileCompleted": false,
      "createdAt": "2026-06-08T09:00:00Z"
    },
    "profileCompleted": false
  }
}
```

| timezone | string | 是 | 例如 `Asia/Shanghai` |
| targetDate | string | 否 | 目标日期，格式 `YYYY-MM-DD` |

## 6.1 获取热量目标调整建议

### 请求方式

```text
GET
```

### URL

```text
/v1/profile/calorie-target-suggestion
```

### 是否需要登录

```text
是
```

### 参数

```json
{
  "headers": {
    "Authorization": "Bearer <accessToken>"
  }
}
```

### 返回结果

该接口只返回建议，不会自动修改当前热量目标。

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "suggest_lower",
    "currentTargetKcal": 1800,
    "suggestedTargetKcal": 1700,
    "changeKcal": -100,
    "reason": "过去 14 天体重下降慢于预期，建议小幅下调每日目标热量",
    "requiresUserConfirmation": true,
    "trendDays": 14,
    "startAverageWeightKg": 80.0,
    "endAverageWeightKg": 79.9,
    "weeklyWeightChangeKg": -0.1
  }
}
```

`status` 可选值：

| 值 | 说明 |
| --- | --- |
| insufficient_data | 体重记录不足，无法判断 |
| not_applicable | 当前目标不处于减脂热量缺口场景 |
| on_track | 体重变化接近预期，暂不建议调整 |
| suggest_lower | 建议下调目标热量 |
| suggest_higher | 建议上调目标热量 |
| at_safety_floor | 已到安全下限，不建议继续下调 |

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| recognitionTaskId | string | 否 | AI 识别任务 ID，手动记录为空 |
| mealDate | string | 是 | 业务日期 |
| mealType | string | 是 | `breakfast`、`lunch`、`dinner`、`snack` |
| sourceType | string | 是 | `manual`、`text`、`photo` |
| rawText | string | 否 | 原始文本 |
| imageUrl | string | 否 | 图片 URL |
| items | array | 是 | 食物明细，至少 1 项 |
| items[].foodId | string | 否 | 本地食物库 ID，来自 `GET /v1/foods/search` 或 `GET /v1/foods/{foodId}` |
| items[].nutritionSource | string | 否 | `food_db`、`ai_estimated`、`user_confirmed`、`user_override`；不传时后端自动推断 |

`nutritionSource` 建议：

| 场景 | 建议值 |
| --- | --- |
| 直接采用本地食物库计算值 | `food_db` |

| 热量与三大营养素 | `proteinG * 4 + carbsG * 4 + fatG * 9` 与 `caloriesKcal` 偏差不能过大 |

## 基础食物库数据维护

以后新增公共基础食物不需要改 Java 代码，直接维护 CSV：

```text
server/src/main/resources/db/seed/food-catalog/foods.csv
server/src/main/resources/db/seed/food-catalog/aliases.csv
server/src/main/resources/db/seed/food-catalog/portions.csv
```

新增一个食物时：

```text
1. 在 foods.csv 追加主数据，包括每 100g 热量、蛋白质、脂肪、碳水。
2. 在 aliases.csv 追加搜索别名，没有别名可以不加。
3. 在 portions.csv 追加默认份量，建议至少 1 条 defaultPortion=true。
4. 启动后端，导入器会自动 upsert 到 food_catalog / food_aliases / food_portions。
```

配置：

```json
{
  "FOOD_CATALOG_SEED_ENABLED": "true",
  "FOOD_CATALOG_SEED_LOCATION": "classpath:db/seed/food-catalog"
}
```

已经上线过的食物不建议从 CSV 删除；需要下线时把 `foods.csv` 里的 `verified` 改为 `false`。

连接信息：

```json
{
  "host": "localhost",
  "port": 5432,
  "database": "leanmate",
  "user": "leanmate",
  "password": "leanmate_local_password"
}
```
