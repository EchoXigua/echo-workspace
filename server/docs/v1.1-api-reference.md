# LeanMate V1.1 后端接口文档

本文给开发、联调和人工验收使用，按接口逐个说明请求方式、URL、参数和返回结果。

机器可读契约仍以 [`../../docs/api/openapi.yaml`](../../docs/api/openapi.yaml) 为准。

## 基础信息

本地服务地址：

```text
http://localhost:8080
```

除登录和刷新 token 外，接口都需要在 Header 中携带访问令牌：

```json
{
  "Authorization": "Bearer <accessToken>"
}
```

统一成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

统一错误响应：

```json
{
  "code": 40101,
  "message": "未登录或登录已过期",
  "data": null
}
```

常见错误码：

| code | 含义 |
| --- | --- |
| 40001 | 参数错误 |
| 40101 | 未登录或 token 无效 |
| 40301 | 无权限访问资源 |
| 40401 | 资源不存在 |
| 40901 | 状态冲突 |
| 50001 | 服务端错误 |
| 50010 | AI 服务暂时不可用 |

## 1. 第三方登录

### 请求方式

```text
POST
```

### URL

```text
/v1/auth/oauth-login
```

### 是否需要登录

```text
否
```

### 参数

local/test 环境支持 mock Apple 登录。`identityToken` 格式为 `mock:<providerUserId>:<email>`。

```json
{
  "body": {
    "provider": "apple",
    "identityToken": "mock:apple-user-1:user@example.com",
    "authorizationCode": null,
    "deviceId": "local-test"
  }
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| provider | string | 是 | 当前支持 `apple`，`google` 保留但 V1.1 不支持 |
| identityToken | string | 是 | Apple identity token；本地可使用 mock token |
| authorizationCode | string | 否 | Apple 授权码，当前可为空 |
| deviceId | string | 否 | 设备标识，最长 128 |

### 返回结果

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "accessToken": "eyJhbGciOi...",
    "refreshToken": "refresh-token-value",
    "tokenType": "Bearer",
    "expiresIn": 3600,
    "user": {
      "id": "11111111-1111-1111-1111-111111111111",
      "nickname": null,
      "avatarUrl": null,
      "status": "active",
      "profileCompleted": false,
      "createdAt": "2026-06-08T09:00:00Z"
    },
    "profileCompleted": false
  }
}
```

| timezone | string | 是 | 例如 `Asia/Shanghai` |
| targetDate | string | 否 | 目标日期，格式 `YYYY-MM-DD` |

## 13. 同步游客本地饮食记录

### 请求方式

```text
POST
```

### URL

```text
/v1/diet/entries/sync-local
```

### 是否需要登录

```text
是
```

### 参数

用于用户登录后，把游客模式保存在本地的 confirmed 饮食记录批量导入后端。后端按当前用户和 `clientLocalId` 幂等处理，同一个本地记录重复提交不会重复入库。

```json
{
  "headers": {
    "Authorization": "Bearer <accessToken>"
  },
  "body": {
    "entries": [
      {
        "clientLocalId": "77777777-7777-7777-7777-777777777777",
        "entry": {
          "recognitionTaskId": null,
          "mealDate": "2026-06-08",
          "mealType": "lunch",
          "sourceType": "manual",
          "rawText": "游客本地午餐",
          "imageUrl": null,
          "items": [
            {
              "id": null,
              "name": "鸡胸肉",
              "quantityText": "150g",
              "weightG": 150,
              "caloriesKcal": 248,
              "proteinG": 46,
              "fatG": 5,
              "carbsG": 0,
              "confidence": null,
              "userEdited": true
            }
          ]
        },
        "createdAt": "2026-06-08T03:00:00Z",
        "updatedAt": "2026-06-08T03:05:00Z"
      }
    ]
  }
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| entries | array | 是 | 待同步本地记录，1 到 200 条 |
| entries[].clientLocalId | string | 是 | 客户端游客模式生成并持久化的本地记录 ID，UUID 格式 |
| entries[].entry | object | 是 | 饮食记录内容，结构同“保存饮食记录”的 body |
| entries[].createdAt | string | 是 | 本地记录创建时间，ISO 8601 |
| entries[].updatedAt | string | 是 | 本地记录更新时间，ISO 8601 |

## 推荐验证顺序

建议按下面顺序验，避免因为没有用户、没有档案或没有数据导致后续接口返回空。

```text
1. POST /v1/auth/oauth-login
2. GET /v1/me
3. GET /v1/profile
4. PUT /v1/profile
5. GET /v1/home/today
6. POST /v1/weights
7. GET /v1/weights
8. POST /v1/diet/recognitions/text
9. GET /v1/diet/recognitions/{taskId}
10. POST /v1/diet/entries
11. GET /v1/diet/entries
12. POST /v1/diet/entries/sync-local
13. GET /v1/home/today
14. POST /v1/reports/daily
15. GET /v1/reports/daily
16. POST /v1/reports/daily/{reportId}/view
17. GET /v1/retention/streak
```
