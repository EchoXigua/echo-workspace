# 后端技术方案检查清单

每个后端技术方案进入实现前，至少检查以下内容。

## 需求范围

- 是否标明对应 PRD 章节。
- 是否写清业务目标。
- 是否写清后端职责。
- 是否写清不做范围。
- 是否列出待确认问题。

## 数据库设计

- 是否列出新增或修改的表。
- 是否引用 `server/docs/database-design.md` 的具体表。
- 是否说明主键、唯一约束、索引。
- 是否说明状态字段和枚举值。
- 是否说明软删除、审计、历史数据保留策略。
- 是否说明事务边界和数据一致性要求。

## API 设计

- 是否列出新增和修改接口。
- 是否引用 `docs/api/openapi.yaml`。
- 是否引用当前版本 API 设计说明。
- 是否说明请求参数、响应数据、错误场景。
- 是否说明客户端是否可以基于接口 Mock 开发。

## 业务规则

- 是否说明核心计算规则。
- 是否说明状态流。
- 是否说明边界条件。
- 是否说明权限和数据归属。
- 是否说明失败降级策略。

## 异步和任务

- 是否说明同步处理还是异步处理。
- 是否说明任务状态。
- 是否说明失败重试。
- 是否说明是否需要 Redis/MQ。V1.1 默认不需要。

## AI 和外部服务

- 是否涉及 AI Provider、对象存储、第三方登录、推送等外部服务。
- 是否说明外部服务失败时的降级方案。
- 是否说明密钥和敏感配置的存储方式。
- 是否说明成本控制、调用频率或超时策略。

## 安全和合规

- 是否说明鉴权要求。
- 是否说明用户是否只能访问自己的数据。
- 是否涉及饮食图片、体重等敏感数据。
- 是否说明日志中不能输出哪些字段。
- 是否涉及用户数据删除、图片保存期限或隐私政策。

## 环境和配置

- 是否需要新增环境变量。
- 是否需要新增对象存储 bucket、AI Key、第三方登录配置。
- 是否影响 local/dev/prod 环境。
- 是否需要更新 `.env.*.example` 或部署文档。

## 埋点和指标

- 是否列出关键事件。
- 是否说明该需求影响哪些成功指标。
- 是否需要更新 `docs/data/events.md` 或 `docs/data/metrics.md`。

## 测试

- 是否覆盖正常流程。
- 是否覆盖参数校验。
- 是否覆盖权限隔离。
- 是否覆盖状态流和异常路径。
- 是否覆盖统计重算或副作用。

## 上线和兼容

- 是否需要数据库迁移脚本。
- 是否需要数据回填。
- 是否影响旧接口兼容。
- 是否需要灰度或开关。
- 是否有回滚风险。
