# LeanMate V1.1 iOS 状态矩阵

Last updated: 2026-06-07

## 文档定位

本文用于指导 iOS V1.1 业务开发前的页面状态设计。

当前版本已锁定第 1、2、3、4、5 批业务：

- Onboarding / 登录占位
- Profile / 用户档案填写
- Mock 登录和档案保存流程
- Home / 首页今日状态
- Diet / 饮食记录入口、文本识别确认、手动保存
- Weight / 体重记录 Sheet 和保存
- Diet / 拍照识别确认和删除确认
- Report / AI 日报页面、生成、查看和失败状态
- Profile / 我的页、数据与计划展示
- Retention / 连续打卡展示和里程碑弹窗

不在本文阶段实现：

- 真实 Apple 登录
- 客户端自行计算 streak
- 里程碑奖励、积分商城或跨会话弹窗去重

Profile Summary、Retention 已在第 5 批按本文格式追加状态矩阵。

## 状态命名约定

ViewModel 状态建议统一使用以下语义，不要求所有 ViewModel 都包含全部状态：

| 状态 | 含义 |
| --- | --- |
| `idle` | 初始或等待用户输入 |
| `loading` | 正在加载或提交 |
| `success` | 当前操作成功 |
| `empty` | 数据为空但不是错误 |
| `error` | 请求或业务失败 |
| `visitor` | 未登录游客态 |
| `profileIncomplete` | 已登录但档案未完成 |

页面实现要求：

- 每个状态只能有一个最强主行动。
- 保存类动作必须防重复点击。
- 失败后保留用户已输入内容。
- 页面不得绕过 `APIClient` / `MockAPIClient` 直接构造业务结果。

## App 路由状态矩阵

| 状态 | 触发条件 | 用户看到什么 | 用户能做什么 | 接口 | Mock 场景 | 验收点 |
| --- | --- | --- | --- | --- | --- | --- |
| coldStart | App 启动，Token 状态未知 | 启动过渡或基础壳 | 等待状态判断 | 读 `TokenStore`，必要时 `GET /v1/me` | 默认成功态 | 不闪现错误页面 |
| visitor | 无 access token | 欢迎页 / 登录占位入口 | mock 登录或游客浏览占位 | 不调业务接口 | visitor | 不允许保存长期数据 |
| authenticatedProfileIncomplete | 有 token，`profileCompleted = false` | 档案填写页 | 填写并保存档案 | `GET /v1/profile`、`PUT /v1/profile` | profileIncomplete | 不进入完整首页 |
| authenticatedProfileCompleted | 有 token，`profileCompleted = true` | 首页今日状态 | 查看今日目标、摄入、剩余、体重、连续天数、饮食摘要、日报摘要 | `GET /v1/me`、`GET /v1/home/today` | success / empty | 不展示 OpenAPI 之外统计 |
| authExpired | token 失效或 401 | 登录过期提示 | 重新登录 | `POST /v1/auth/refresh` 可失败 | error | 清理失效 token，不停留在假登录态 |

第 1 批开发约束：

- 可以使用 Mock 登录模拟 token 写入。
- 不接真实 Apple 登录按钮能力。
- 首页只作为路由落点或占位，不实现统计、饮食、体重、日报内容。

- 第 1 批不实现真实 Apple 登录授权、隐私协议页或账号删除能力。

计划文件：

- `Features/Home/HomeView.swift`
- `Features/Home/HomeViewModel.swift`
- `Features/Home/HomeEmptyView.swift`
- `Features/Home/VisitorHomeBanner.swift`

对应接口：

- `GET /v1/home/today`

| 状态 | 用户看到什么 | 用户能做什么 | 接口 | Mock 场景 | 验收点 |
| --- | --- | --- | --- | --- | --- |
| loading | 首页加载状态，保留底部导航 | 等待或切换底部导航 | `GET /v1/home/today` | success / empty | 不显示空白页，不展示旧数据 |
| visitor | 登录后同步记录提示、有限首页提示 | 登录；点击需要持久化动作时引导登录 | 不调业务保存接口 | visitor | 游客不能保存档案、饮食、体重或日报查看状态 |
| profileIncomplete | 完成档案提示 | 去填写档案 | `GET /v1/home/today` 返回 `profileCompleted = false` 或路由判断 | profileIncomplete | 不展示假的热量目标 |
| empty | 今天还没有饮食、体重或日报数据的空态 | 去记录 | `GET /v1/home/today` | empty | 不展示假的热量、体重或日报数据 |
| loaded | 今日目标热量、已摄入、剩余、营养、当前体重、连续天数、饮食摘要、日报摘要 | 切换底部导航或进入后续记录入口 | `GET /v1/home/today` | success | 全部数值以后端返回为准 |
| error | 可理解错误提示 | 重试；401 时返回登录 | `GET /v1/home/today` | error | 不停留在假登录态，不吞错误 |

开发处理：

- HomeViewModel 负责状态流转和错误映射，View 不直接调用 API。
- empty 状态可以基于后端返回的无记录数据判断，但 UI 不展示热量、体重或日报假数据。
- loaded 状态展示 `date`、`calorieTargetKcal`、`caloriesInKcal`、`remainingCaloriesKcal`、`proteinG`、`fatG`、`carbsG`、`currentWeightKg`、`streakDays`、`reportSummary`、`foodEntries`。
- `streakDays` 只展示后端返回值，不在客户端重算。
- 第 2 批不实现饮食记录、体重记录、AI 日报详情、拍照识别或文本识别确认页。

- `文字解析确认 / 食卡卡风`

计划文件：

- `Features/Diet/DietEntryView.swift`
- `Features/Diet/DietEntryViewModel.swift`

对应接口：

- `POST /v1/diet/recognitions/photo`
- `POST /v1/diet/recognitions/text`
- `GET /v1/diet/recognitions/{taskId}`
- `POST /v1/diet/entries`
- `DELETE /v1/diet/entries/{entryId}`

| 状态 | 用户看到什么 | 用户能做什么 | 接口 | Mock 场景 | 验收点 |
| --- | --- | --- | --- | --- | --- |
| idle | 拍照、文本、手动三个入口和今日餐次/日期 | 选择记录方式；输入文本或手动食物 | 不调接口 | success | 不默认创建记录 |
| photoEditing | 照片选择区、照片预览、确认前提示 | 选择或重新选择照片；开始拍照识别 | 不调接口，开始识别时调 `POST /v1/diet/recognitions/photo` | success | 权限只在选择照片时触发，空照片不调接口 |
| textEditing | 文本输入区 | 输入一句自然语言饮食描述，选择餐次和日期 | 不调接口 | success | 空文本不允许提交 |
| recognizing | 识别中状态 | 等待或取消等待 | `POST /v1/diet/recognitions/text` 或 `POST /v1/diet/recognitions/photo` | success / recognitionRunning | 防重复点击，不重复创建任务 |
| recognitionRunning | 任务仍在 pending/running | 手动刷新识别结果或改为手动记录 | `GET /v1/diet/recognitions/{taskId}` | recognitionRunning | 不阻塞用户继续手动保存 |

| confirmation | 可编辑识别结果、总热量估算 | 修改食物名称、数量、重量、热量和营养；确认保存 | 不调接口 | success | 用户确认前不计入正式记录 |
| manualEditing | 手动录入食物名称、数量、重量、热量和营养 | 保存手动饮食记录 | 不调接口 | success | 必填字段本地校验失败不调 API |
| saving | 保存中 | 等待 | `POST /v1/diet/entries` | success | 保存按钮防重复点击 |
| saveSucceeded | 保存成功提示 | 返回首页或继续记录 | `POST /v1/diet/entries` | success | 使用后端返回结果，不客户端重算统计 |
| saveFailed | 保存失败提示 | 修改或重试 | `POST /v1/diet/entries` | error | 失败后保留输入和确认内容 |
| deleteConfirming | 删除二次确认弹窗 | 取消或确认删除 | 确认前不调接口 | success | 删除前必须二次确认 |
| deleting | 删除中状态 | 等待 | `DELETE /v1/diet/entries/{entryId}` | success | 防重复删除 |
| deleteSucceeded | 删除成功提示 | 继续记录或返回首页 | `DELETE /v1/diet/entries/{entryId}` | success | 以后端重新计算统计为准 |
| deleteFailed | 删除失败提示 | 关闭提示或重试删除 | `DELETE /v1/diet/entries/{entryId}` | error | 不清空已保存记录引用 |

字段验收：

| 字段 | 输入范围 | UI 处理 |
| --- | --- | --- |
| mealDate | 业务日期 | 默认今天，可通过 DatePicker 调整 |
| mealType | `breakfast` / `lunch` / `dinner` / `snack` | 分段选择或等价控件 |
| text | 1-1000 字符 | 空文本不调识别接口 |
| food name | 1-128 字符 | 手动和确认保存必填 |
| quantityText | 可选，最长 128 字符 | 失败后保留 |
| weightG / caloriesKcal / proteinG / fatG / carbsG | 可选数字 | 非数字本地提示 |

开发处理：

- `DietEntryViewModel` 负责文本识别、识别任务刷新、确认草稿映射和保存状态。
- `SaveFoodEntryRequest.sourceType` 必须按入口传 `photo`、`text` 或 `manual`，不使用 OpenAPI 之外字段。
- 识别失败后允许用原文本预填手动记录，不丢失用户输入。
- 保存成功后可提示用户返回首页；客户端不自行刷新首页统计。
- 删除只调用 `deleteDietEntry(id:)`，删除前必须有二次确认。

| idle | 初始状态 | 等待页面加载 | 不调接口 | 任意 | 不显示过期数据 |
| loading | 我的页加载状态，保留底部导航 | 等待或切换导航 | `GET /v1/me`、`GET /v1/profile`、`GET /v1/retention/streak` | success / empty | 不显示空白页 |
| visitor | 登录后查看计划提示 | 返回登录 | 不调业务保存接口 | visitor | 游客不读取或保存长期数据 |
| profileIncomplete | 完成档案提示 | 去填写档案 | `GET /v1/me` 或 `GET /v1/profile` 返回档案未完成 | profileIncomplete | 不展示假的目标或打卡数据 |
| loaded | 用户摘要、当前体重、目标体重、每日目标、BMI、BMR、连续打卡和里程碑标签 | 刷新；切换导航 | `GET /v1/me`、`GET /v1/profile`、`GET /v1/retention/streak` | success | 目标和 streak 全部以后端返回为准 |
| emptyStreak | 我的页正常展示，连续打卡为 0，里程碑均未达成 | 继续记录饮食或体重 | `GET /v1/retention/streak` | empty | 不伪造连续打卡天数 |
| error | 我的页加载失败提示 | 重试；401 时重新登录 | 任一接口失败 | error | 401 清理 token；其他错误保留可重试入口 |
| milestoneEligible | 已达成最高里程碑弹窗 | 关闭弹窗 | 使用 `milestones.achieved = true` 的返回项 | success | 只展示后端返回的达成里程碑，不客户端生成奖励 |
| milestoneDismissed | 弹窗关闭后回到我的页 | 继续查看或切换导航 | 不调接口 | success | 当前 ViewModel 生命周期内不重复展示同一最高里程碑 |

字段验收：

| 字段 | 来源 | UI 处理 |
| --- | --- | --- |
| nickname / avatarUrl | `CurrentUser` | nickname 为空时展示默认用户名；当前不新增头像上传 |
| bmi / bmrKcal / dailyCalorieTargetKcal | `UserProfile` | 只展示后端返回值，不重新计算 |
| currentWeightKg / targetWeightKg | `UserProfile` | 作为档案数据展示，不在我的页编辑 |
| currentDays / longestDays / lastActiveDate | `Streak` | 只展示后端返回值 |
| milestones | `Streak.milestones` | 只展示 3、7、30、100 天等 OpenAPI 返回的里程碑 |

开发处理：

- `ProfileSummaryViewModel` 负责加载 `me/profile/streak`、状态映射、401 清理 token 和里程碑弹窗选择。
- 里程碑弹窗只从后端返回的已达成 `milestones` 中选择最高里程碑展示；关闭后不倒退展示更低里程碑。
- 当前不扩展 `LocalStore` 做跨会话弹窗去重，避免引入 OpenAPI 之外的 retention 同步语义。
- 第 5 批不实现奖励、积分商城、真实 Apple 登录、体重趋势计算或客户端 streak 计算。
