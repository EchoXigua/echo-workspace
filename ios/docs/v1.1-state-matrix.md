# LeanMate V1.1 iOS 状态矩阵

Last updated: 2026-06-07

## 文档定位

本文用于指导 iOS V1.1 业务开发前的页面状态设计。

当前版本只锁定第 1 批业务：

- Onboarding / 登录占位
- Profile / 用户档案填写
- Mock 登录和档案保存流程

不在本文阶段实现：

- 首页完整统计
- 饮食记录
- 体重记录
- AI 日报
- 真实 Apple 登录

后续 Home、Diet、Weight、Report、Retention 开工前，应按本文格式追加各自状态矩阵。

## 状态命名约定

ViewModel 状态建议统一使用以下语义，不要求所有 ViewModel 都包含全部状态：

| 状态 | 含义 |
| --- | --- |
| `idle` | 初始或等待用户输入 |
| `loading` | 正在加载或提交 |
| `success` | 当前操作成功 |
| `empty` | 数据为空但不是错误 |
| `error` | 请求或业务失败 |
| `visitor` | 未登录游客态 |
| `profileIncomplete` | 已登录但档案未完成 |

页面实现要求：

- 每个状态只能有一个最强主行动。
- 保存类动作必须防重复点击。
- 失败后保留用户已输入内容。
- 页面不得绕过 `APIClient` / `MockAPIClient` 直接构造业务结果。

## App 路由状态矩阵

| 状态 | 触发条件 | 用户看到什么 | 用户能做什么 | 接口 | Mock 场景 | 验收点 |
| --- | --- | --- | --- | --- | --- | --- |
| coldStart | App 启动，Token 状态未知 | 启动过渡或基础壳 | 等待状态判断 | 读 `TokenStore`，必要时 `GET /v1/me` | 默认成功态 | 不闪现错误页面 |
| visitor | 无 access token | 欢迎页 / 登录占位入口 | mock 登录或游客浏览占位 | 不调业务接口 | visitor | 不允许保存长期数据 |
| authenticatedProfileIncomplete | 有 token，`profileCompleted = false` | 档案填写页 | 填写并保存档案 | `GET /v1/profile`、`PUT /v1/profile` | profileIncomplete | 不进入完整首页 |
| authenticatedProfileCompleted | 有 token，`profileCompleted = true` | 首页占位或后续首页 | 进入主流程 | `GET /v1/me`、后续 `GET /v1/home/today` | success | 第 1 批只进入首页占位，不做完整首页 |
| authExpired | token 失效或 401 | 登录过期提示 | 重新登录 | `POST /v1/auth/refresh` 可失败 | error | 清理失效 token，不停留在假登录态 |

第 1 批开发约束：

- 可以使用 Mock 登录模拟 token 写入。
- 不接真实 Apple 登录按钮能力。
- 首页只作为路由落点或占位，不实现统计、饮食、体重、日报内容。

## Onboarding 状态矩阵

对应设计稿：

- `欢迎启动页 / 食卡卡风`

计划文件：

- `Features/Onboarding/OnboardingView.swift`
- `Features/Onboarding/OnboardingViewModel.swift`

| 状态 | 用户看到什么 | 用户能做什么 | 接口 | Mock 场景 | 验收点 |
| --- | --- | --- | --- | --- | --- |
| idle | 欢迎页、品牌信息、主行动按钮 | 点击开始 / mock 登录 | 不直接调业务接口 | visitor | 主行动清晰，不展示无关功能 |
| loggingIn | 登录按钮 loading | 等待 | `POST /v1/auth/oauth-login`，第 1 批用 Mock | success / profileIncomplete | 禁用重复点击 |
| loginSucceededProfileIncomplete | 登录成功但未完成档案 | 自动进入档案填写 | `GET /v1/profile` 可由 Mock 返回 | profileIncomplete | 进入 ProfileSetup |
| loginSucceededProfileCompleted | 登录成功且已完成档案 | 自动进入首页占位 | `GET /v1/me` 或 token payload | success | 不再要求重复填写档案 |
| loginFailed | 登录失败提示 | 重试 | Mock error | error | 错误文案可理解，保留主行动 |
| visitorPreview | 游客浏览占位 | 查看有限提示 / 返回登录 | 不调业务保存接口 | visitor | 游客不能保存档案或记录 |

开发处理：

- `OnboardingViewModel` 只负责入口状态和 mock 登录流程。
- 路由跳转交给 `AppRouter` 或上层容器，不在 View 内写硬编码分支。
- 第 1 批不实现真实 Apple 登录授权、隐私协议页或账号删除能力。

## Profile Setup 状态矩阵

对应设计稿：

- `Onboarding - 唯一问题 / 食卡卡风`

计划文件：

- `Features/Profile/ProfileSetupView.swift`
- `Features/Profile/ProfileSetupViewModel.swift`

对应接口：

- `GET /v1/profile`
- `PUT /v1/profile`
- `GET /v1/me`

| 状态 | 用户看到什么 | 用户能做什么 | 接口 | Mock 场景 | 验收点 |
| --- | --- | --- | --- | --- | --- |
| loadingProfile | 档案加载状态 | 等待 | `GET /v1/profile` | profileIncomplete / success | 不显示空白页 |
| editingEmpty | 空表单或默认值 | 输入档案 | 不提交接口 | profileIncomplete | 所有必填字段可完成 |
| editingExisting | 已有档案值 | 修改并保存 | 不提交接口 | success | 已有值正确回填 |
| localValidationFailed | 表单错误提示 | 修改错误字段 | 不提交接口 | 任意 | 不发起无效请求 |
| saving | 保存中 | 等待 | `PUT /v1/profile` | success | 禁用重复保存 |
| saveSucceeded | 保存成功、展示目标结果 | 进入首页占位 | `PUT /v1/profile` 返回 `ProfilePayload` | success | 展示或可访问 BMI/BMR/每日热量目标 |
| saveFailed | 保存失败提示 | 修改或重试 | `PUT /v1/profile` | error | 保留用户输入 |
| authExpired | 登录过期提示 | 返回登录 | `GET /v1/profile` 或 `PUT /v1/profile` 返回 401 | error | 不继续提交档案 |

字段验收：

| 字段 | 输入范围 | UI 处理 |
| --- | --- | --- |
| 性别 | `male` / `female` / `unknown` | 分段选择或等价控件 |
| 年龄 | 1-120 | 数字输入，本地校验 |
| 身高 | 50-250 cm | 数字输入，本地校验 |
| 当前体重 | 20-300 kg | 数字输入，本地校验 |
| 目标体重 | 20-300 kg | 数字输入，本地校验 |
| 活动水平 | OpenAPI 枚举 | 选项控件，不允许自由文本 |
| timezone | 必填 | 默认使用设备时区，保存到请求 |
| targetDate | 可选 | 第 1 批可不暴露复杂日期选择 |

开发处理：

- 本地校验通过后才调用 `PUT /v1/profile`。
- 客户端展示后端返回的 `bmi`、`bmrKcal`、`dailyCalorieTargetKcal`，不自行覆盖计算结果。
- 第 1 批不允许手动修改每日热量目标。

## 首页占位状态矩阵

第 1 批只需要首页占位作为路由落点，不实现 Home 业务。

| 状态 | 用户看到什么 | 用户能做什么 | 接口 | Mock 场景 | 验收点 |
| --- | --- | --- | --- | --- | --- |
| placeholder | 首页占位或基础壳 | 确认已进入主流程 | 不调 `GET /v1/home/today` | success | 不展示假的热量、体重、日报数据 |

开发处理：

- 首页完整状态矩阵在第 2 批 Home 开工前补充。
- 不为了占位页提前实现首页业务 ViewModel。

## Mock 场景要求

第 1 批业务至少需要覆盖：

| Mock 场景 | 用途 |
| --- | --- |
| success | 已登录且档案完成，进入首页占位 |
| profileIncomplete | 已登录但档案未完成，进入档案填写 |
| error | 登录、获取档案或保存档案失败 |
| visitor | 未登录游客入口 |

如现有 `MockAPIClient.Scenario` 不能直接覆盖 visitor，可在业务 ViewModel 中通过无 token 状态表达 visitor，不扩展 OpenAPI。

## 测试验收

第 1 批业务建议补充 ViewModel 测试：

| 测试 | 验收点 |
| --- | --- |
| mock 登录后档案未完成 | 路由进入 ProfileSetup |
| mock 登录后档案已完成 | 路由进入首页占位 |
| Profile 本地校验失败 | 不调用保存接口 |
| Profile 保存成功 | 状态进入 success，路由可进入首页占位 |
| Profile 保存失败 | 保留输入并展示 error |
| 保存中重复点击 | 只发起一次保存动作 |

UI 层验收：

- 欢迎页和档案页在 iPhone 常见尺寸下文字不截断。
- 表单错误提示不遮挡输入。
- loading、error、success 状态可见。

## 后续模块追加规则

后续每个 Feature 开工前，必须在本文追加对应状态矩阵：

- Home：loaded、empty、visitor、profileIncomplete、error。
- Diet：entry、recognition running、recognition failed、confirmation、delete confirmation。
- Weight：sheet editing、saving、saved、error、trend empty。
- Report：pending、generated、viewed、failed、notEnoughData。
- Retention：normal、milestoneUnlocked、alreadyViewed。
