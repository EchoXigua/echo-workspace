# ADR 0001：MVP 阶段采用模块化单体

## 状态

已接受

## 背景

LeanMate V1.1 的核心目标是验证持续记录和 AI 日报的产品价值。当前功能范围包括用户档案、饮食记录、体重记录、热量统计、AI 日报和连续打卡。未来会扩展 AI 周报/月报、AI 教练、行为洞察和个性化减脂模型。

这些能力存在明显业务域边界，但 MVP 阶段尚未证明用户留存和核心价值，不适合过早拆分微服务。

## 决策

服务端采用一个 Spring Boot 应用，但内部按业务域分模块：

- user
- diet
- weight
- stats
- report
- retention
- ai

模块之间通过应用服务和明确的数据模型协作。禁止把所有业务逻辑堆到一个全局 service 包中。

## 后果

优点：

- 部署和调试简单。
- 开发速度快。
- 事务处理直接。
- 后续仍可按模块拆出独立服务。

代价：

- 需要自律维护模块边界。
- 需要避免跨模块直接访问彼此内部表和实现类。
- 当 AI 任务、报表生成、推送提醒变重后，需要引入异步任务或拆出 worker。

## 后续触发条件

满足以下条件时再考虑拆分：

- AI 识别或日报生成明显拖慢主 API。
- 定时任务、推送、报表生成占用大量资源。
- 单体应用部署频率和稳定性无法满足多团队协作。
- 某个模块需要独立扩缩容。
