# ADR 0002：V1.1 开始接入后端

## 状态

已接受

## 背景

LeanMate V1.1 包含用户档案、饮食记录、体重记录、AI 日报和连续打卡。后续确定会继续做 iOS 和 Android，产品做大后会考虑 Web 端。

V1.1 曾考虑过只使用 iPhone 本地存储和 iCloud/CloudKit 完成数据保存与同步。

## 决策

V1.1 从一开始接入 LeanMate 后端。后端作为核心业务入口和数据真相来源。

iCloud/CloudKit 不作为主后端。后续如果需要，可以作为 iOS 端本地备份、草稿同步或 Apple 生态体验增强。

## 原因

- AI 图片识别、文本解析和日报生成需要后端代理，避免 AI Key 暴露在 App 内。
- 后续 Android 无法直接复用 iCloud 数据。
- 未来 Web 端需要统一账号、接口和数据模型。
- 留存率、日报阅读率、记录次数、建议采纳率等指标需要统一采集。
- AI 周报、月报、行为洞察、风险预警依赖长期结构化数据。

## 后果

优点：

- 跨端一致性更好。
- AI 成本、安全、限流和日志可控。
- 后续产品演进空间更大。
- 数据资产可以持续沉淀。

代价：

- V1.1 初始开发量增加。
- 需要部署、数据库、运维和接口联调。
- 客户端需要处理网络失败和本地草稿。

## 执行方式

- 后端采用 Spring Boot 3.x 模块化单体。
- 数据库优先使用 PostgreSQL。
- OpenAPI 作为跨端接口契约。
- App 不直接调用 AI Provider。
- V1.1 不做复杂离线同步，跨端数据以后端为准。
